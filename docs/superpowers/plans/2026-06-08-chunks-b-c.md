## Task C1: fixed July 1 → June 30 wrap window in `_maybe_wrap_sim_year_end`

**Files:** Modify `src/captest/captest.py` (`_maybe_wrap_sim_year_end`); Modify `tests/test_captest.py` (`TestAutoWrapSim`).

The 60-day-from-year-boundary trigger and the snapshot/restore logic stay. Only the window computation changes: instead of deriving the window from the measured test dates (with leap-day clipping), use a fixed July 1 → June 30 window, years derived from `sim_year` (so it is `1989-07-01` → `1990-06-30` for the standard pvsyst case, where `load_pvsyst` normalizes data to 1990). This yields a contiguous 8760-row sim year centered on the Jan 1 boundary regardless of where the measured test falls.

- [ ] **Step 1: Add the 8760-row assertion test (failing only if the window is wrong)**

Add to `TestAutoWrapSim` in `tests/test_captest.py`:
```python
    def test_wrapped_frame_has_8760_rows(self):
        # Fixed July 1 1989 -> June 30 1990 window is a full, contiguous year.
        meas_idx = pd.date_range("2023-11-01", "2023-12-15", freq="h")
        sim = _hourly_typical_year(1990)
        ct = _ct_with(meas_idx, sim.index)
        ct._maybe_wrap_sim_year_end()
        assert len(ct.sim.data) == 8760
        assert ct.sim.data.index.min() == pd.Timestamp("1989-07-01 00:00")
        assert ct.sim.data.index.max() == pd.Timestamp("1990-06-30 23:00")
        assert ct.sim.data.index.is_monotonic_increasing
        assert not ct.sim.data.index.has_duplicates
```
